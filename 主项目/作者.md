## 启动后端

先释放 8000 端口（可选），再进入虚拟环境启动：

```bash
cd backend
# 可选：结束占用 8000 的进程（macOS/Linux）
for pid in $(lsof -ti tcp:8000 2>/dev/null); do kill -15 "$pid" 2>/dev/null || true; done
source .venv/bin/activate
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

或使用仓库根目录的 **start_backend.sh**（Ubuntu/服务端：`chmod +x start_backend.sh && ./start_backend.sh`；会检查/创建 `.venv` 并安装依赖）。

- **Ubuntu 图形界面双击启动**：使用应用根目录 **0401多了AI-主/多乐AI-后端启动.desktop**（`Exec` 指向 **0401多了AI-主/run_backend_gui.sh**）。首次右键 **允许启动**。若移动了仓库，请编辑 `.desktop` 里 **Exec=** 的绝对路径。
- **命令行启动同一桌面项**（在应用根目录）：`cd .../0401多了AI-主 && gio launch ./多乐AI-后端启动.desktop`（不要用 `gtk-launch 全路径`，`gtk-launch` 只认已安装应用的 ID）。

## 环境自检（可选）

- **ffmpeg**：非 Windows 下默认用系统 PATH；Mac 可用 `brew install ffmpeg`。本机开发前可执行 `ffmpeg -version`。
- **Playwright**：安装依赖后需 `cd backend && .venv/bin/playwright install chromium`（或 `playwright install`）。
- **Ollama / CUDA**：仅在配置 `LLM_PROVIDER=ollama` 或 GPU 版 ASR 时需要；未装则保持 mock/CPU 即可。
