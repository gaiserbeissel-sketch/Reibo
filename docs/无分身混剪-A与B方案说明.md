# 无分身混剪：功能说明与 A / B 方案区别

本文档整理此前在计划中讨论的 **无分身混剪** 功能要点，以及转场「不重复」规则的 **A 方案** 与 **B 方案** 的含义与区别。

---

## 一、功能（当时讨论的需求）

| 项目 | 说明 |
|------|------|
| **适用场景** | 勾选「无分身混剪」：成片以口播音频为主时间轴，**不**使用步骤 3 分身视频；可用若干 **插入素材**（视频/图）做画面。 |
| **镜头节奏** | 素材按时间轴 **打碎** 成镜头：**5 秒 → 6 秒 → 5 秒 → 6 秒 …** 交替；由 `settings.compose_media_only_sec_first` / `compose_media_only_sec_second` 配置（默认 5/6）。有分身时的素材段仍用 `compose_cut_media_sec_*`（默认 7/6）。 |
| **短素材** | 若某条素材 **不足 5 秒**，**不丢弃**：通过 **循环/拼接**（`stream_loop` + `trim` 或按时间偏移多段取样）把该素材 **填满** 当前槽位，而不是忽略。 |
| **长素材** | 同一文件需 **按时间往后切**（多段时 `seek`/`trim` 起点递增），避免每段都从 0 秒开始重复同一画面。 |
| **段内特效** | 与现有逻辑一致：插入段可使用 `INSERT_MEDIA_EFFECT_VFS` 等随机滤镜（需与「不重复」策略对齐）。 |
| **转场** | 镜头间使用 **xfade 类转场**，每次 **随机**；**同一种转场名**在何种范围内 **不重复出现两次**，由 **A 或 B** 定义（见下）。 |

**实现说明（已落地）**：[`backend/app/video.py`](../backend/app/video.py) 已允许 `talking_head_url` 为空且 `insert_media_urls` 非空时走 `_compose_cut_broadcast`；无分身时镜间转场使用 **`_xfade_transition_drawer()`（方案 B，全局不重复）**；**减少重复画面**主要靠同一源文件的 **`seek` 时间指针顺延** + 不足一段时用 **`stream_loop` + trim** 拼满槽位。

---

## 二、A 方案与 B 方案：在管什么

两者都只讨论 **「镜头接镜头」之间的转场名字**（如 `fade`、`wipeleft`），不讨论口播、字幕。

| 对比项 | **A 方案（按每个素材文件记账）** | **B 方案（整条片子一本总账）** |
|--------|----------------------------------|--------------------------------|
| **记账单位** | **每个上传的 mp4（或每个素材路径）各一本「已用转场」清单**。 | **整支成片** 只有 **一本** 「已用转场」清单。 |
| **核心规则** | 同 **一个源文件** 多次出现在时间轴上时，**尽量不要** 对 **该文件** 反复使用同一种转场名；**换另一个文件** 可以重新用同一种转场名。 | **不区分** 来自哪个文件；**只要** 某种转场名在片子里用过一次，**后面尽量别再用**（直到多种转场轮完再洗牌）。 |
| **典型区别** | 甲已用过 `fade`，乙 **第一次** 参与转场时 **仍可选用 `fade`**（乙自己的清单为空）。 | 甲已用过 `fade`，后面 **无论** 接乙还是接甲，**都不能** 再用 `fade`（总账已记）。 |

### 一句话对照

- **A**：「**同一个上传的视频文件**」内部，同一种转场尽量不重复。  
- **B**：「**整支成品视频**」里，同一种转场尽量不重复。

### 和你原话的对应

- 「**同一个视频** 同一个特效不可以出现两次」若指 **某一个上传的 mp4** → 选 **A**。  
- 若指 **整支成片** → 选 **B**。

---

## 三、功能与 A/B 的关系

- **功能**（5/6 秒节奏、短素材打碎拼装、段内随机特效、转场随机）是 **成片流水线规则**。  
- **A / B** 只解决 **转场名字** 在 **「按文件」** 还是 **「按全片」** 上 **如何不重复**，属于 **转场策略**，不影响口播时长、槽位秒数等业务规则本身。

实现时：先落地 **无分身 + 插入素材** 的切播与 **5/6 秒**、**seek/循环**；再在 **xfade 链** 上挂 **A 或 B** 的选名器（可为每个 `Path` 维护 `set`/shuffle 池，或复用全局 `_xfade_transition_drawer` 思路）。

---

## 四、选型结论（与「重复画面少」的关系）

- **转场 A / B**：二者都不直接改变「同一帧出现多少次」；**减少素材画面重复**依赖 **seek 顺延** 与 **循环拼段**。  
- **转场策略**：已采用 **B**（与现有 `_xfade_transition_drawer` 一致，整条成片内转场名优先不重复）。  
- 若日后需要改为 **A**（按每个源文件单独记转场），需在 `_xfade_chain_media_segments` 处改为按 `Path` 分池选名。
